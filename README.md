# sw_algorithm

## Complexity
### 시간 복잡도 (프로그램의 예상 소요 시간)
* 데이터의 크기(N)와 구현하는 로직에 따라 결정
* Big-O 표기법: 구현하는 로직과 N에 따라 연산 횟수를 계산하여 표시 (N은 최고 차수만 고려, 계수 무시)

✅ N의 크기에 따라 몇 번의 연산을 수행 되는가?
**1억 번 연산 = 1초**

### 공간 복잡도 (메모리 사용량)

* 사용되는 변수에 따라 결정
* 선언된 변수들 (배열, 자료 구조)의 크기를 계산하면 프로그램 공간 복잡도 계산 가능
* python은 데이터를 하나의 객체로 보기 때문에 c++ 대비 유동적 (c++은 데이터 자료형 별로 크기가 정해져있음)
*pro 검정 서버로 test 시, 단순 int형 객체의 배열은 2천만가지 잡을 수 있음*

📝 pro 검정은 50개 test case를 주어진 시간 안에 프로그램이 동작해야하고, 메모리 256MB까지 허용 (stack 메모리는 1MB)  
✔️ python 3 ~ 15초 정도의 시간이 주어진다.  
✔️ 전체 test case 중 데이터 크기(N)가 최대로 들어오는 test case는 1~2개 뿐  
✔️ 최대 N이 들어 올 때를 생각하여, O(1억) = 1초를 넘어가는지 기준으로 통과 기준을 판별하자.  
❗️ pro 검정은 input case에 따라 횟수가 달라지는 경우가 많으므로 worst/average case를 고려하자.  
❗️ 지역 변수 특히 재귀 함수 사용 시 주의해야 한다.

> stack 메모리란  
> pro 검정에서 코드영역/데이터영역/heap/stack 합쳐 256MB 넘어가면 안된다. 
> * 코드 영역: 실행할 프로그램 코드, 프로그램 시작될 때 로드되고, 실행중에 거의 변하지 않는다.  
> * 데이터 영역 (전역/정적 변수): 프로그램 시작할 때 메모리 확보, 끝날 때까지 유지  
> * 힙 영역: 동적으로 생성되는 메모리, 런타임 크기가 결정되고 python 대부분 객체는 여기 저장 (리스트, 딕셔너리, 객체)  
>*  스택 영역 (지역 변수/매개 변수/함수 호출): 함수 끝나면 자동으로 제거

 🌟 그렇다면 설계는 어떻게 하면 될까?
 1. 입력 크기를 확인한다. 최대로 들어오는 N의 크기는 얼마인가를 보고 시간 복잡도를 판단한다. 

    |N 크기|가능한 시간복잡도|
    |--|--|
    |10|O(N!)|
    |20|O(2^N)|
    |1000|O(N²)|
    |100000|O(N log N)|
    |1000000|O(N)|
2. 입력 크기를 보고 알고리즘 종류를 선택한다.
~~~
그래프 연결 -> DFS/BFS
최단 거리 -> BFS/다익스트라
~~~
3. 선택한 알고리즘이 시간 제한 안에 들어오는지 확인한다. (시간복잡도 계산)
4. 공간복잡도를 확인한다. 
~~~
python int, bool는 약 28bytes, list 원소는 8 bytes, 문자열 문자 1~4 bytes

e.g.) arr = [1,2,3] 
list 원소 = 8 bytes (포인터)
int 객체 = 28 bytes
list에 int 하나 ≈ 36 bytes

1 byte = 8 bit
1 KB = 1024 bytes
1 MB = 1024 KB
1 GB = 1024 MB
~~~
5. 코드 구조를 정하면서 구현 방법을 결정한다.
~~~
문제: 그래프 탐색 문제
1) 그래프 인접 리스트 생성
2) visited 배열 생성
3) DFS 또는 BFS 수행
4) 결과 출력
~~~
